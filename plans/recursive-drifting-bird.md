# カラーシミュレーターの色選択変更プラン

## Context

カラーシミュレーター（http://localhost:8001/simulator）の色選択機能を変更し、`/カラー一覧/一覧/` フォルダにある新しいカラー（30色）を選択可能にする。

現在のシミュレーターは16色の固定パレットを使用している。新しいカラーはC1.png画像から抽出されたスウォッチ画像（swatch_01_01.png〜swatch_03_10.png）として存在する。

## 実装方針

### 1. 色データの抽出

`/カラー一覧/一覧/` フォルダにある30個のスウォッチPNGファイルから、以下の情報を抽出する：

- カラーコード（品番）：C1.png画像に記載されている「09-1024」等のコード
- 色名：C1.png画像に記載されている日本語色名
- HEX値：スウォッチ画像の代表色（中央部分のピクセル値から取得）

### 2. カラーパレットデータの更新

**ファイル**: `/simulator/js/palette.js`

現在の16色の定義を新しい30色に置き換える。

変更後の形式：
```javascript
const COLOR_PALETTE = [
  { code: '09-1024', name: 'ブラック', hex: '#1a1a1a' },
  { code: '09-1025', name: 'チャコール', hex: '#2d2d2d' },
  // ... 合計30色
];
```

### 3. 初期色設定の更新

**ファイル**: `/simulator/js/simulator.js`

INITIAL_COLORS内の色コードを、新しいカラーコードに合わせて更新する。

## 変更対象ファイル

1. `/simulator/js/palette.js` - メインのカラーパレット定義
2. `/simulator/js/simulator.js` - 初期色設定（INITIAL_COLORS）

## 実装手順

1. スウォッチ画像から色情報（HEX値）を抽出するスクリプトを作成
2. C1.png画像からカラーコードと色名を取得（OCRまたは手動入力）
3. 抽出したデータを元に、新しいCOLOR_PALETTE配列を作成
4. palette.jsを更新
5. simulator.jsのINITIAL_COLORSを新しいコードに合わせて更新
6. 動作確認

## 検証方法

1. シミュレーターを起動（http://localhost:8001/simulator）
2. カラーパレットに30色が表示されていることを確認
3. 各パーツ（A/B/C/D）の色変更が正常に動作することを確認
4. URLパラメータによる状態保存・復元が正常に動作することを確認
